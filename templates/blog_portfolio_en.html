Building a Portfolio with Flask: Combining Simplicity and Performance

PythonFlaskVercel Published on Dec 11, 2025 • 5 min read

In the world of web development, a portfolio is much more than a simple showcase; it's the first project a recruiter analyzes. For my own platform, I chose to move away from "turnkey" solutions like WordPress to build a custom architecture using Flask.

Why Flask?

Flask is a Python micro-framework that prioritizes freedom. Unlike Django, it doesn't impose a rigid structure. This allowed me to create clean routing and manage my content dynamically while keeping the application extremely lightweight and fast to load.

Frontend Challenges and Design

For the design, I opted for a modern Glassmorphism style, reflecting a technical and contemporary identity. The main challenge was making the interface fully responsive. I used CSS Grid and Flexbox to ensure the user experience is seamless, whether on a 27-inch screen or a smartphone.

Deployment and CI/CD

A modern developer must know how to deploy. My portfolio is hosted on Vercel, with a continuous deployment pipeline linked to my GitHub repository. Every "push" to the main branch automatically triggers an update, ensuring my site is always in sync with my latest work.

Conclusion

This project is just the foundation. The Flask architecture I've implemented will allow me to easily add new features, such as a comment system or a "Lab" section for my future technical experiments.
